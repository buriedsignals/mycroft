02 Local model

Local models.

Pick the model that matches the work. Qwen 9B is the Mycroft daily driver. Spotlight needs a larger local model for full investigations.

Browser-only hardware probe. No data leaves this page.
-Hardware recommendation

system_profiler SPHardwareDataType | grep -E 'Memory|Chip|Cores'
One-click local: the setup script downloads the model you picked into ~/models/ and registers it with Goose's built-in Local Inference (llama.cpp, embedded — no separate server to start). When Goose opens, the model is already selected.

The 9B handles both Mycroft daily-driver work and Spotlight investigations — same vault, same skills, same model. The 27B is for journalists with 32 GB+ Macs who want maximum reasoning depth and don't mind 3-5× slower responses.

07 The script

Review.
Then run.

The exact bash that will land in your downloads folder. Your keys are baked in plaintext — review before running. It writes the Goose-facing Mycroft profile to ~/.config/goose/mycroft/, updateable source to ~/.local/share/goose/mycroft/source/, and an updater job for repo updates.

Download. Run. Client-side only — your API keys are embedded in the file, nothing is sent over the network.
Downloaded. Extract the ZIP anywhere, then double-click mycroft-setup.command. The script configures Goose with Mycroft at ~/.config/goose/mycroft and uses the vault paths you selected. On macOS, if Apple says it could not verify the file, choose Done, then open System Settings → Privacy & Security → Open Anyway. After the first approved run, plain double-click works. The agent setup ZIP contains the same API keys in its local manifest so an agent can perform setup without asking you to paste secrets into chat.
